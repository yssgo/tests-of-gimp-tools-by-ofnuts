ofn-dial-marks

Purpose

This script creates dial marks (regularly spaced disk sectors) as paths.

Operation

The script is started from Filters>Render>Paths>Dial marks

Parameters:

  • Center X, Center Y: the coordinates of the center of the dial.
  • Inner radius, Outer radius: the distance from the center of the two extremities of the marks.
  • Number of marks: the number of marks to generate.
  • Mark width: the width of the marks, in degrees.
  • Start angle: the angle of the center of the first mark. 0° is 0300 hours, 90° is 0600 hours, etc. Negative values (for instance, -90° for 1200 hours) are accepted.
  • Mark shape: Polygonal is made of straight lines, Rounded uses circle arcs for the inner and outer edges.
  • Direction: indicates the order in which the marks are generated: clockwise or counter-clockwise.
  • Generated path(s): determines if the output is a single path or if there is one path per mark.

Usage notes

  • The Direction and Generated path(s) parameters are mostly useful when using the script to generate animations (such as "spinners").
  • The paths are generated bottom-up (the first path is the one at the bottom of the list in the Paths dialog). This is coherent with other scripts (for instance, ofn-stroke-fill-paths) that will use paths to render frames for an animation. In other words the Direction parameter sets the rotation direction of the final animation.
