ofn-rotate-layer

This script is used to create multiple rotated copies of a layer. A typical use case is rotating animations (but it has other uses).

This script is called via the menu entry: Layer>Transform>Multi-rotate...

Parameters

  • Direction: determines if the rotation is clockwise or counter-clockwise.
  • Angle: the angle between each copy of the layer. For the mathematically inclined, negative angles are not accepted, and the only way to change the rotation direction is the Direction parameter. If Angle is left to 0., then it is assumed to be 360/count.
  • Count: the number of copies to produce. If left to 0, then it is assumed to be 360/angle.
  • Auto-center: if set, the layer is rotated around its own center (which may not be the image center)
  • CenterX, CenterY: the coordinates of the center of rotation if the Auto-center function is not used.
  • Layer name: a pattern to generate the layer name for the rotated copies (see below)

Layer name pattern

The layer name pattern specifies how the layers are named using specific layer data. In the pattern, names in braces are replaced by the values of these data for the layer. For instance, a pattern of Rotation-{AngleInt}째 will produce the name Rotation-45째 for the rotated copy which is 45째 from the source.

The available values are:

  • sourceName: the name of the source layer
  • count: the total number of layers produced (typically used in "X of Y" names)
  • direction: "CW" or "CCW"
  • num0: the index number of the rotated layer (counted from 0, as comupters count)
  • num1: the index number of the rotated layer (counted from 1, as humans count)
  • angleInt: the angle of rotation for the layer, rounded to the nearest integer
  • angleDec: the angle of rotation for the layer, with a decimal part.

The name pattern is actually directly used as a Python format specification, so modifiers can be used.

  • The most useful modifier for numbers is the ability to define a minimal length with zero-padding. This is done by adding :0Nd to the variable name in the braces, for instance {angleInt:03d} will insert the angle on at least three digits, with zeroes added to the left if necessary: 045, 090, 115.
  • For numbers with a decimal part the number of digits after the decimal point can be specied as :.Nf, for instance {angleDec:.2f} would generate values such as 45.00, 112.25

Usage notes

  • The first copy corresponds to a rotation of 0째, and is therefore identical to the source layer. This is done to have all layers consistently named.
  • Since this first copy is included in the count, you may need to specify one more copy in some cases (but the typical use case of generating copies for a rotation animation doesn't require this).
