ofn-file-next

Purpose

This script reduces the time it takes to manually edit a series of images by automating the sequence

  • Save current picture,
  • Quit current picture
  • Open the file selector and search the next picture in a potentially long list of files
  • Open the next file in sequence

Usage

The script appears in the menu as two entries: FileðŸ šOpen next imageðŸ šIn lexical order, and FileðŸ šOpen next imageðŸ šIn numerical order.

You normally open the first file using the regular FileðŸ šOpen, and when done with that one use the script functions to go to the next image.

This script is best used by assigning one of the two functions to a shortcut:

  • No need to hunt the function in menus
  • If you assign only one order type to a keystroke, you remove the chance to switch to another sequence by mistake.

The functions appear in the shortcut editor as python-fu-ofn-file-next-lexical and python-fu-ofn-file-next-numerical respectively.

Lexical v.s. Numerical

Given the list:

20.jpg
jkl.jpg

The lexical order (case-independent for the plugin) is:

11.jpg
145.jpg
20.jpg
abc.jpg
blank.png
def.jpg
GHI.jpg
jkl.jpg
Test003.jpg
Test02.jpg
Test1.jpg
Test40-01.jpg
Test4.jpg

So, 145.jpg is processed before 20.jpg, and likewise Test1.jpg is processed after Test02.jpg that itself comes after Test003.jpg

In numerical order, the plugins looks for digits at the end of the file name, converts that part to a number, and sorts using this number the files that have the same non-numerical prefix, so in this set of files there are two distinct sequences:

  • One with an empty prefix:

    11.jpg
20.jpg
145.jpg

  • and one with the Test prefix

    Test1.jpg
Test02.jpg
Test003.jpg
Test4.jpg
